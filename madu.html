Pipeline de Dados

O Pipeline de Dados é o fluxo contínuo e automatizado que move e processa dados brutos desde a fonte até um destino onde podem ser consumidos. As etapas principais são:

1. Coleta (Ingestion) Captura de dados de diversas fontes (logs de servidor, APIs, bancos de dados, streaming).
2. Processamento (Processing) Limpeza, transformação, filtragem e agregação. Pode ser em lote (batch) ou streaming.
3. Armazenamento (Storage) Persistência dos dados processados ou brutos (Data Lakes ou Data Warehouses).
4. Análise (Analysis) Aplicação de estatística, Machine Learning e BI para extrair inteligência.

Tecnologias Chave

O ecossistema é vasto, mas três tecnologias fundamentam o Big Data moderno:

Hadoop

Framework open-source para processamento distribuído. Foca em armazenar (HDFS) e processar (MapReduce) dados de forma tolerante a falhas em clusters.

Spark

Mecanismo unificado e veloz. Realiza cálculos na memória RAM (até 100x mais rápido que o MapReduce). Suporta streaming e Machine Learning.

NoSQL

Bancos de dados flexíveis sem esquema rígido (Not Only SQL). Feitos para escalabilidade horizontal. Exemplos: MongoDB, Redis, Cassandra.

Data Lakes vs. Data Warehouses

Ambos armazenam dados em escala, mas com propósitos distintos:

Característica Data Warehouse (DW) Data Lake (DL)
Estrutura Estruturada
(Esquema na escrita)		 Bruta/Não Estruturada
(Esquema na leitura)
Objetivo Relatórios, BI e consultas rápidas. Análise profunda, Machine Learning e Data Science.
Qualidade Dados limpos e transformados. Dados brutos, sem filtros.
Usuários Analistas de Negócios, Gerentes. Cientistas e Engenheiros de Dados.

O Data Warehouse armazena dados processados para relatórios específicos. Já o Data Lake armazena tudo, e a estrutura é definida apenas quando o dado é lido (Schema-on-Read).
